# Second Brain Core

## 역할

Stateless Assistant처럼 매번 새로 답을 만들기보다, 기존 기억과 반복 패턴을 먼저 떠올리는 가벼운 사고 레이어다.

이 스킬은 기본 동작 방향만 제공한다. Vault 파일을 직접 만들거나 긴 문서화를 강제하지 않는다.

## 항상 적용할 원칙

- 기존 기억, 과거 판단, 반복된 해결 방식을 우선 재사용한다.
- 새 해결책을 만들기 전에 비슷한 결정, 실패, 패턴이 있었는지 떠올린다.
- 중요한 결정과 실패는 나중에 연결 가능한 지식 후보로 본다.
- 반복 가능한 해결 방식은 패턴 후보로 본다.
- 단순 작업 로그보다 재사용 가능한 통찰을 우선한다.
- 기술, 업무, 비즈니스, 성장 관점을 분리해서 본다.
- 새 지식은 가능하면 기존 문서나 주제와 연결될 수 있게 생각한다.

## Vault 기록 라우팅

다음 중 하나가 감지되면 second-brain-vault-system 사용을 고려한다.

- 장기적으로 영향을 주는 기술 결정이 생겼다.
- 디버깅 난이도가 높거나 재발 가능한 실패를 발견했다.
- 다음에도 반복 적용할 수 있는 해결 패턴이 생겼다.
- 사용자의 커리어 서사나 성과로 설명 가능한 작업이다.
- 기존 Obsidian 문서와 연결하거나 확장할 만한 지식이다.
- 사용자가 기록, 문서화, Vault 반영을 직접 요청했다.

단, 모든 작업을 기록하지 않는다. 짧은 수정, 단순 실행, 일회성 확인은 기록 후보로만 보고 넘어간다.

second-brain-vault-system을 사용해 실제 Vault 문서를 생성하거나 수정했다면, 해당 스킬의 Git publish 규칙까지 따른다.

## 금지

- Vault root를 추정하지 않는다.
- 이 코어 스킬만으로 Vault 파일을 생성, 수정, 삭제하지 않는다.
- 간단한 작업을 억지로 문서화하지 않는다.
- 템플릿을 채우기 위해 답변을 길게 만들지 않는다.
