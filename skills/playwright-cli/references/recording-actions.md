# 동작 녹화 규칙

사용자가 브라우저에서 수행한 동작을 나중에 다시 실행할 수 있게 저장할 때 적용하는 규칙이다.

## 트리거
- 사용자가 `new record`
- 사용자가 `이거 동작 저장하자`
- `지금 하는 작업 녹화하자`
- `이 행동 저장해`

이런 식으로 말하면 새로운 recorder 세션을 시작한다.

## 원칙
- 항상 새 녹화를 시작한다. 직전 녹화 결과를 이어붙이지 않는다.
- `new record`가 오면 저장 이름보다 URL 확인이 우선이다.
- URL을 받기 전에는 recorder를 열지 않는다.
- recorder는 사용자가 `종료`라고 말할 때까지 유지한다.
- 사용자가 이름을 이미 줬으면 그 이름으로 저장한다.
- 사용자가 이름을 안 줬으면 임시 파일명으로 먼저 저장하고, 녹화가 끝난 뒤 이름을 물어본다.
- 녹화는 기본적으로 `npx playwright codegen`을 사용한다.
- 녹화된 동작 재생은 기본적으로 `scripts/run-recorded-action.sh`로 수행한다.
- 재생 시에는 항상 보이는 브라우저(`--headed`)를 사용한다.
- 재생 중 실패하면 기본적으로 최대 3회까지 재시도한다.
- 재시도는 처음부터가 아니라 마지막 성공 다음 단계부터 이어서 진행한다.
- 결과물은 `playwright-cli/recordings` 아래에 둔다.

## 저장 방식
- 임시 파일 예시:
  - `recordings/pending-20260331-111500.spec.js`
  - `recordings/pending-20260331-111500.auth.json`
- 최종 이름이 정해지면 kebab-case로 바꿔 저장한다.
  - 예: `요양급여의뢰서` -> `yoyang-geubyeo-uiroe`

## 진행 순서
1. 사용자가 `new record`라고 하면 URL을 물어본다.
2. 사용자가 URL을 주면 그 URL로 recorder를 연다.
3. 사용자가 recorder 브라우저에서 직접 동작한다.
4. 사용자가 `종료`라고 할 때까지 recorder 세션을 유지한다.
5. recorder 종료 후 생성된 파일을 확인한다.
6. 이름이 없으면 사용자에게 저장 이름을 묻는다.
7. 최종 이름으로 파일을 정리하고, 필요하면 관련 reference 문서나 실행 스크립트로 연결한다.
8. 사용자가 실행을 요청하면 `scripts/run-recorded-action.sh <spec-file>` 형태로 재생한다.
9. 재생이 중단되면 같은 script가 checkpoint 파일을 보고 마지막 성공 단계 다음부터 이어서 재생한다.

## 후처리
- 녹화 결과가 다른 화면 흐름으로 저장됐으면 잘못된 녹화로 보고 폐기하거나 다시 녹화한다.
- 저장된 코드는 그대로 두지 말고, 필요하면 불필요한 클릭이나 잘못된 selector를 정리한다.
