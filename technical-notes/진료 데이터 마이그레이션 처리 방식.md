# 진료 데이터 마이그레이션 처리 방식

## 목적

외부 원천 데이터를 표준 진료 데이터로 옮길 때, 실제 스키마명이나 특정 회사명을 문서에 남기지 않고 재사용 가능한 처리 방식만 정리한다.

이 노트는 실행 SQL 보관용이 아니라, 어떤 산출물을 만들고 어떤 방식으로 이관 리스크를 줄였는지 다음 작업에서 재사용하기 위한 기준이다.

## 작업 산출물

이번 작업의 핵심 산출물은 두 가지다.

- 원천 데이터를 표준 진료 데이터로 이관하기 위한 실행 프로그램을 구성했다.
- ![[Screenshot 2026-05-21 at 4.20.53 PM.png]]
- 
- 상병 데이터와 처방 데이터를 변환 임시 영역에 적재하는 전용 프로시저를 작성했다.

실행 프로그램은 매번 SQL을 수동으로 찾아 실행하는 방식의 부담을 줄이고, 마이그레이션 실행 진입점을 하나로 모으기 위한 도구다. 실제 접속 정보나 특정 운영 환경 정보는 문서에 남기지 않는다.

![진료 데이터 마이그레이션 실행 프로그램](../attachments/진료%20데이터%20마이그레이션%20실행%20프로그램.svg)

프로그램에서는 원천 데이터 확인, 영역별 이관 실행, 검증 결과 확인을 한 흐름으로 묶었다. 운영자가 실행 순서를 기억해서 SQL을 개별로 실행하는 구조가 아니라, 정해진 메뉴에서 필요한 단계를 실행하고 결과를 확인하는 구조로 정리했다.

프로시저는 원천 조회, 값 변환, 순번 채번, 임시 적재까지의 반복 가능한 단위를 고정하기 위해 작성했다. 최종 반영 전 검증을 남겨두기 위해 바로 최종 영역에 insert하지 않고 임시 영역에 먼저 적재하는 구조로 잡았다.

## 처리 흐름

원천 데이터는 실행 프로그램을 통해 정해진 절차로 실행하고, 프로시저는 바로 최종 테이블에 넣지 않고 먼저 변환용 임시 영역에 적재한다. 이후 검증을 거쳐 최종 영역으로 반영한다.

```text
실행 프로그램 -> 이관 프로시저 호출 -> 원천 조회 -> 변환/매핑 -> 임시 적재 -> 중복/키 검증 -> 최종 반영
```

이 구조를 둔 이유는 세 가지다.

- 원천 컬럼과 최종 컬럼의 의미가 1:1로 맞지 않는 경우가 있어 중간 변환 결과를 확인해야 한다.
- 최종 데이터에 기존 이력 또는 일부 선적재 데이터가 남아 있을 수 있어 바로 insert하면 키 충돌이 날 수 있다.
- 처방명, 수가명처럼 기준정보를 조인해야 하는 항목은 조인 증폭 여부를 별도로 검증해야 한다.

## 공통 처리 기준

한글 데이터는 원천 DB에서 먼저 샘플링해 인코딩 깨짐 여부를 확인한다. DB TO DB 이관 중 한글이 깨지면 변환 함수를 적용해 원천 값을 확인한 뒤, 실제 적재 기준을 잡는다.

예시는 실제 스키마명, 기관명, 회사명을 제거한 형태로 남긴다.

```sql
-- 원천 한글 컬럼 인코딩 확인 예시
SELECT UTL_I18N.RAW_TO_NCHAR(
           UTL_RAW.CAST_TO_RAW(src.DIAGNOSIS_NAME),
           'KO16MSWIN949'
       ) AS CONVERTED_NAME
FROM `상병_원천` src
WHERE ROWNUM < 10;
```

순번은 신규 데이터만 보고 1부터 채번하지 않는다. 최종 영역에 이미 존재하는 최대 순번을 조회하고, 그 이후 번호부터 신규 행에 부여한다.

```text
신규 순번 = 기존 최종 순번의 최대값 + 원천 변환 결과의 행 번호
```

이 방식은 일부 데이터가 이미 들어간 상태에서 추가 이관을 수행해도 기존 키를 덮거나 충돌시키지 않기 위한 처리다.

## 상병 데이터 처리

상병 데이터는 전용 이관 프로시저를 작성해 접수 단위로 묶고 순번을 부여한다.

처리 과정은 다음 기준을 따른다.

- 원천의 진단 구분, 주상병/부상병 구분, 진료과, 진료의, 진단일시를 최종 진료 기준 값으로 변환한다.
- 같은 접수 안에서 여러 상병이 존재할 수 있으므로 접수 단위 그룹 안에서 순번을 채번한다.
- 최종 영역에 이미 같은 접수의 상병이 있으면 기존 최대 순번 이후부터 추가한다.
- 임시 적재 후에는 접수 단위 키와 순번 조합이 중복되지 않는지 확인한다.

프로시저 작성의 목적은 상병 원천 구조를 표준 진료 상병 구조로 일관되게 변환하고, 기존 최종 데이터가 있는 상태에서도 키 충돌 없이 추가 적재할 수 있게 만드는 것이다.

상병 이관 프로시저의 핵심 쿼리 형태는 다음과 같다. 실제 스키마명과 원천 테이블명만 제거하고, 테이블명은 역할이 보이도록 한글로 치환했다. `:co_cd`, `:div_cd`는 실제 실행 환경 값으로 치환되는 자리다.

```sql
DROP PROCEDURE IF EXISTS `상병_마이그레이션`;

DELIMITER $$

CREATE PROCEDURE `상병_마이그레이션`()
BEGIN
    INSERT INTO `상병_임시` (
        `CO_CD`,
        `DIV_CD`,
        `RCPN_NB`,
        `ILNS_SQ`,
        `MAIN_SUB_TYPE_FG`,
        `DGNS_FG`,
        `DGNS_CD`,
        `DGNS_NM`,
        `DGNS_ENM`,
        `DGNS_DEPT_CD`,
        `DGNS_EMP_CD`,
        `DGNS_DT`,
        `DGNS_RGST_DTS`,
        `MDCR_DEPT_CD`,
        `MDCR_EMP_CD`,
        `DGNS_SITE_FG`,
        `RO_YN`,
        `POA_FG`
    )
    SELECT
        numbered.`CO_CD`,
        numbered.`DIV_CD`,
        numbered.`RCPN_NB`,
        numbered.`ILNS_SQ`,
        numbered.`MAIN_SUB_TYPE_FG`,
        numbered.`DGNS_FG`,
        numbered.`DGNS_CD`,
        numbered.`DGNS_NM`,
        numbered.`DGNS_ENM`,
        numbered.`DGNS_DEPT_CD`,
        numbered.`DGNS_EMP_CD`,
        numbered.`DGNS_DT`,
        numbered.`DGNS_RGST_DTS`,
        numbered.`MDCR_DEPT_CD`,
        numbered.`MDCR_EMP_CD`,
        numbered.`DGNS_SITE_FG`,
        numbered.`RO_YN`,
        numbered.`POA_FG`
    FROM (
        SELECT
            base.`CO_CD`,
            base.`DIV_CD`,
            base.`RCPN_NB`,
            COALESCE(existing.`MAX_ILNS_SQ`, 0)
                + ROW_NUMBER() OVER (
                    PARTITION BY base.`CO_CD`, base.`DIV_CD`, base.`RCPN_NB`
                    ORDER BY base.`DGNS_DT`, base.`DGNS_RGST_DTS`
                ) AS `ILNS_SQ`,
            base.`MAIN_SUB_TYPE_FG`,
            base.`DGNS_FG`,
            base.`DGNS_CD`,
            base.`DGNS_NM`,
            base.`DGNS_ENM`,
            base.`DGNS_DEPT_CD`,
            base.`DGNS_EMP_CD`,
            base.`DGNS_DT`,
            base.`DGNS_RGST_DTS`,
            base.`MDCR_DEPT_CD`,
            base.`MDCR_EMP_CD`,
            base.`DGNS_SITE_FG`,
            base.`RO_YN`,
            base.`POA_FG`
        FROM (
            SELECT
                :co_cd AS `CO_CD`,
                :div_cd AS `DIV_CD`,
                CONCAT(src.`A`, src.`B`, src.`C`, src.`D`) AS `RCPN_NB`,
                CASE src.`D`
                    WHEN 'M' THEN 'M'
                    WHEN 'S' THEN 'S1'
                    WHEN 'T' THEN 'S2'
                    WHEN 'U' THEN 'S3'
                    WHEN 'V' THEN 'S4'
                    WHEN 'W' THEN 'S5'
                    ELSE NULL
                END AS `MAIN_SUB_TYPE_FG`,
                CASE src.`G`
                    WHEN '1' THEN '1'
                    WHEN '2' THEN '2'
                    WHEN '3' THEN '2'
                    WHEN '5' THEN '3'
                    ELSE NULL
                END AS `DGNS_FG`,
                src.`H` AS `DGNS_CD`,
                src.`I` AS `DGNS_NM`,
                src.`I` AS `DGNS_ENM`,
                src.`C` AS `DGNS_DEPT_CD`,
                src.`L` AS `DGNS_EMP_CD`,
                src.`A` AS `DGNS_DT`,
                src.`N` AS `DGNS_RGST_DTS`,
                src.`C` AS `MDCR_DEPT_CD`,
                src.`L` AS `MDCR_EMP_CD`,
                CASE src.`Q`
                    WHEN 'B' THEN 'B'
                    WHEN 'L' THEN 'L'
                    WHEN 'R' THEN 'R'
                    ELSE NULL
                END AS `DGNS_SITE_FG`,
                CASE src.`Q`
                    WHEN 'RO' THEN 'Y'
                    ELSE NULL
                END AS `RO_YN`,
                CASE src.`S`
                    WHEN 'Y' THEN 'Y'
                    WHEN 'N' THEN 'N'
                    WHEN 'U' THEN 'U'
                    WHEN 'W' THEN 'W'
                    WHEN 'E' THEN 'E'
                    ELSE NULL
                END AS `POA_FG`
            FROM `상병_원천` src
        ) base
        LEFT JOIN (
            SELECT
                `CO_CD`,
                `DIV_CD`,
                `RCPN_NB`,
                MAX(`ILNS_SQ`) AS `MAX_ILNS_SQ`
            FROM `상병_최종`
            GROUP BY `CO_CD`, `DIV_CD`, `RCPN_NB`
        ) existing
          ON existing.`CO_CD` = base.`CO_CD`
         AND existing.`DIV_CD` = base.`DIV_CD`
         AND existing.`RCPN_NB` = base.`RCPN_NB`
    ) numbered;

    SELECT ROW_COUNT() AS migrated_rows;
END$$

DELIMITER ;
```

상병 적재 후 검증 쿼리는 다음 형태로 둔다.

```sql
-- 임시 영역 내부 키 중복 확인
SELECT
    t.`CO_CD`,
    t.`DIV_CD`,
    t.`RCPN_NB`,
    t.`ILNS_SQ`,
    COUNT(*) AS cnt
FROM `상병_임시` t
GROUP BY
    t.`CO_CD`,
    t.`DIV_CD`,
    t.`RCPN_NB`,
    t.`ILNS_SQ`
HAVING COUNT(*) > 1;

-- 최종 영역과 키 충돌 확인
SELECT
    t.`CO_CD`,
    t.`DIV_CD`,
    t.`RCPN_NB`,
    t.`ILNS_SQ`
FROM `상병_임시` t
JOIN `상병_최종` m
  ON m.`CO_CD` = t.`CO_CD`
 AND m.`DIV_CD` = t.`DIV_CD`
 AND m.`RCPN_NB` = t.`RCPN_NB`
 AND m.`ILNS_SQ` = t.`ILNS_SQ`;
```

## 처방 데이터 처리

처방 데이터도 전용 이관 프로시저를 작성해 처방 번호 단위로 묶고 순번을 부여한다.

처리 과정은 다음 기준을 따른다.

- 원천 처방에는 지시 성격의 코드나 특수 문자가 섞일 수 있으므로, 처방코드와 처방명 처리 기준을 분리한다.
- 처방명은 원천 문자열을 그대로 쓰기보다 기준정보의 유효기간에 맞는 명칭을 조인해 사용한다.
- 같은 처방 번호 안에서 여러 처방 행이 존재할 수 있으므로 처방 번호 단위 그룹 안에서 순번을 채번한다.
- 최종 영역에 이미 같은 처방 번호의 데이터가 있으면 기존 최대 순번 이후부터 추가한다.
- 기준정보 조인 시 유효기간이 겹치면 원천 1건이 여러 건으로 늘어날 수 있으므로, 실행 전에 중복 매칭 여부를 확인한다.

프로시저 작성의 목적은 처방코드, 처방명, 처방일자, 진료과, 처방자, 투여 관련 값처럼 의미가 다른 원천 컬럼을 표준 처방 구조로 변환하고, 기준정보 조인까지 포함해 반복 실행 가능한 이관 단위를 만드는 것이다.

처방 이관 프로시저의 핵심 쿼리 형태는 다음과 같다. 실제 스키마명과 원천 테이블명만 제거하고, 테이블명은 역할이 보이도록 한글로 치환했다. `:co_cd`, `:div_cd`는 실제 실행 환경 값으로 치환되는 자리다.

```sql
DROP PROCEDURE IF EXISTS `처방_마이그레이션`;

DELIMITER $$

CREATE PROCEDURE `처방_마이그레이션`()
BEGIN
    INSERT INTO `처방_임시` (
        `CO_CD`,
        `DIV_CD`,
        `PRSC_NB`,
        `PRSC_SQ`,
        `RCPN_NB`,
        `PRSC_CD`,
        `PRSC_NM`,
        `PRSC_ENM`,
        `PRSC_DT`,
        `PRSC_RGST_DTS`,
        `MDCR_DEPT_CD`,
        `MDCR_EMP_CD`,
        `PRSC_DEPT_CD`,
        `PRSC_EMP_CD`,
        `PRSC_NOTM`,
        `PRSC_NODY`,
        `PRSC_DOSG`,
        `IOTM_CD`,
        `CMPT_FG`,
        `MAIN_SUB_TYPE_FG`,
        `PRSC_PRGR_STAT`
    )
    SELECT
        numbered.`CO_CD`,
        numbered.`DIV_CD`,
        numbered.`PRSC_NB`,
        numbered.`PRSC_SQ`,
        numbered.`RCPN_NB`,
        numbered.`PRSC_CD`,
        numbered.`PRSC_NM`,
        numbered.`PRSC_ENM`,
        numbered.`PRSC_DT`,
        numbered.`PRSC_RGST_DTS`,
        numbered.`MDCR_DEPT_CD`,
        numbered.`MDCR_EMP_CD`,
        numbered.`PRSC_DEPT_CD`,
        numbered.`PRSC_EMP_CD`,
        numbered.`PRSC_NOTM`,
        numbered.`PRSC_NODY`,
        numbered.`PRSC_DOSG`,
        numbered.`IOTM_CD`,
        numbered.`CMPT_FG`,
        numbered.`MAIN_SUB_TYPE_FG`,
        numbered.`PRSC_PRGR_STAT`
    FROM (
        SELECT
            base.`CO_CD`,
            base.`DIV_CD`,
            base.`PRSC_NB`,
            COALESCE(existing.`MAX_PRSC_SQ`, 0)
                + ROW_NUMBER() OVER (
                    PARTITION BY base.`CO_CD`, base.`DIV_CD`, base.`PRSC_NB`
                    ORDER BY base.`PRSC_DT`, base.`PRSC_RGST_DTS`
                ) AS `PRSC_SQ`,
            base.`RCPN_NB`,
            base.`PRSC_CD`,
            base.`PRSC_NM`,
            base.`PRSC_ENM`,
            base.`PRSC_DT`,
            base.`PRSC_RGST_DTS`,
            base.`MDCR_DEPT_CD`,
            base.`MDCR_EMP_CD`,
            base.`PRSC_DEPT_CD`,
            base.`PRSC_EMP_CD`,
            base.`PRSC_NOTM`,
            base.`PRSC_NODY`,
            base.`PRSC_DOSG`,
            base.`IOTM_CD`,
            base.`CMPT_FG`,
            base.`MAIN_SUB_TYPE_FG`,
            base.`PRSC_PRGR_STAT`
        FROM (
            SELECT
                converted.`CO_CD`,
                converted.`DIV_CD`,
                converted.`PRSC_NB`,
                converted.`RCPN_NB`,
                converted.`PRSC_CD`,
                ref.`PRSC_NM` AS `PRSC_NM`,
                ref.`SUGA_ENM` AS `PRSC_ENM`,
                converted.`PRSC_DT`,
                converted.`PRSC_RGST_DTS`,
                converted.`MDCR_DEPT_CD`,
                converted.`MDCR_EMP_CD`,
                converted.`PRSC_DEPT_CD`,
                converted.`PRSC_EMP_CD`,
                converted.`PRSC_NOTM`,
                converted.`PRSC_NODY`,
                converted.`PRSC_DOSG`,
                converted.`IOTM_CD`,
                converted.`CMPT_FG`,
                converted.`MAIN_SUB_TYPE_FG`,
                converted.`PRSC_PRGR_STAT`
            FROM (
                SELECT
                    :co_cd AS `CO_CD`,
                    :div_cd AS `DIV_CD`,
                    CONCAT(SUBSTRING(src.`A`, 3, 6), src.`B`, src.`D`) AS `PRSC_NB`,
                    CONCAT(src.`A`, src.`B`, src.`C`, src.`D`) AS `RCPN_NB`,
                    src.`E` AS `PRSC_CD`,
                    src.`F` AS `PRSC_DT`,
                    CONCAT(src.`F`, src.`H`) AS `PRSC_RGST_DTS`,
                    src.`C` AS `MDCR_DEPT_CD`,
                    src.`J` AS `MDCR_EMP_CD`,
                    src.`I` AS `PRSC_DEPT_CD`,
                    src.`J` AS `PRSC_EMP_CD`,
                    src.`M` AS `PRSC_NOTM`,
                    src.`N` AS `PRSC_NODY`,
                    src.`O` AS `PRSC_DOSG`,
                    src.`P` AS `IOTM_CD`,
                    CASE src.`S`
                        WHEN 'N' THEN '2'
                        WHEN 'M' THEN '3'
                        ELSE '1'
                    END AS `CMPT_FG`,
                    CASE src.`D`
                        WHEN 'M' THEN 'M'
                        WHEN 'S' THEN 'S1'
                        WHEN 'T' THEN 'S2'
                        WHEN 'U' THEN 'S3'
                        WHEN 'V' THEN 'S4'
                        WHEN 'W' THEN 'S5'
                        ELSE NULL
                    END AS `MAIN_SUB_TYPE_FG`,
                    src.`20` AS `PRSC_PRGR_STAT`
                FROM `처방_원천` src
            ) converted
            LEFT JOIN `처방명_기준` ref
              ON converted.`CO_CD` = ref.`CO_CD`
             AND converted.`DIV_CD` = ref.`DIV_CD`
             AND converted.`PRSC_CD` = ref.`SUGA_CD`
             AND ref.`SUGA_APLY_DT` <= converted.`PRSC_DT`
             AND ref.`SUGA_END_DT` >= converted.`PRSC_DT`
        ) base
        LEFT JOIN (
            SELECT
                `CO_CD`,
                `DIV_CD`,
                `PRSC_NB`,
                MAX(`PRSC_SQ`) AS `MAX_PRSC_SQ`
            FROM `처방_최종`
            GROUP BY `CO_CD`, `DIV_CD`, `PRSC_NB`
        ) existing
          ON existing.`CO_CD` = base.`CO_CD`
         AND existing.`DIV_CD` = base.`DIV_CD`
         AND existing.`PRSC_NB` = base.`PRSC_NB`
    ) numbered;

    SELECT ROW_COUNT() AS migrated_rows;
END$$

DELIMITER ;
```

처방 적재 후 검증 쿼리는 다음 형태로 둔다.

```sql
-- 임시 영역 내부 키 중복 확인
SELECT
    t.`CO_CD`,
    t.`DIV_CD`,
    t.`PRSC_NB`,
    t.`PRSC_SQ`,
    COUNT(*) AS cnt
FROM `처방_임시` t
GROUP BY
    t.`CO_CD`,
    t.`DIV_CD`,
    t.`PRSC_NB`,
    t.`PRSC_SQ`
HAVING COUNT(*) > 1;

-- 최종 영역과 키 충돌 확인
SELECT
    t.`CO_CD`,
    t.`DIV_CD`,
    t.`PRSC_NB`,
    t.`PRSC_SQ`
FROM `처방_임시` t
JOIN `처방_최종` m
  ON m.`CO_CD` = t.`CO_CD`
 AND m.`DIV_CD` = t.`DIV_CD`
 AND m.`PRSC_NB` = t.`PRSC_NB`
 AND m.`PRSC_SQ` = t.`PRSC_SQ`;

-- 기준정보 기간 조인 중복 매칭 확인
SELECT
    converted.`CO_CD`,
    converted.`DIV_CD`,
    converted.`PRSC_CD`,
    converted.`PRSC_DT`,
    COUNT(*) AS matched_count
FROM (
    SELECT
        :co_cd AS `CO_CD`,
        :div_cd AS `DIV_CD`,
        src.`E` AS `PRSC_CD`,
        src.`F` AS `PRSC_DT`
    FROM `처방_원천`
) converted
JOIN `처방명_기준` ref
  ON converted.`CO_CD` = ref.`CO_CD`
 AND converted.`DIV_CD` = ref.`DIV_CD`
 AND converted.`PRSC_CD` = ref.`SUGA_CD`
 AND ref.`SUGA_APLY_DT` <= converted.`PRSC_DT`
 AND ref.`SUGA_END_DT` >= converted.`PRSC_DT`
GROUP BY
    converted.`CO_CD`,
    converted.`DIV_CD`,
    converted.`PRSC_CD`,
    converted.`PRSC_DT`
HAVING COUNT(*) > 1;
```

## 경과기록 데이터 처리

경과기록은 원천과 최종 반영 대상만 확인된 상태로 보고, 임시 구조와 컬럼 매핑은 별도로 확정해야 한다.

확정 전에는 다음을 먼저 정리해야 한다.

- 원천 본문의 단위가 접수, 진료일, 작성자 중 무엇을 기준으로 묶이는지 확인한다.
- 최종 영역의 순번 또는 문서 식별자 생성 기준을 정한다.
- 본문 데이터의 줄바꿈, 서식, 특수문자 보존 범위를 정한다.
- 임시 적재 후 최종 반영 전 검증 쿼리를 정의한다.

## 실행 전 검증

실행 전에는 원천 건수, 필수 컬럼 누락, 한글 인코딩, 기준정보 조인 증폭, 최종 키 충돌 가능성을 먼저 확인한다.

특히 다음 검증은 생략하지 않는다.

- 원천 전체 건수와 변환 후 임시 적재 건수가 의도대로 일치하는지 확인한다.
- 주요 식별자와 일자 값이 비어 있는 행을 확인한다.
- 한글 컬럼 샘플을 직접 확인한다.
- 임시 영역 내부에서 최종 키로 사용할 조합이 중복되지 않는지 확인한다.
- 최종 영역과 비교했을 때 신규 순번이 기존 키와 충돌하지 않는지 확인한다.
- 기준정보 조인이 원천 건수를 늘리지 않는지 확인한다.

## 반복 실행 시 주의점

프로시저나 실행 객체를 제거해도 임시 영역에 적재된 데이터가 자동으로 사라지는 것은 아니다. 반복 실행 전에는 임시 영역 초기화, 실행 이력 관리, 중복 방지 조건 중 하나를 명확히 정해야 한다.

이미 한 번 적재한 원천을 다시 실행하면, 순번 충돌은 피하더라도 같은 의미의 데이터가 다른 순번으로 중복 생성될 수 있다. 따라서 재실행은 항상 다음 중 하나를 먼저 결정한 뒤 수행한다.

- 임시 영역을 비우고 다시 적재한다.
- 원천 기준 중복 방지 조건을 추가한다.
- 실행 회차를 별도 이력으로 남기고 후속 정제 대상으로 관리한다.

## 재사용 기준

이 방식은 원천과 최종 구조가 다르고, 기존 최종 데이터와 신규 이관 데이터가 섞일 수 있는 진료 데이터 마이그레이션에 적용한다.

핵심은 SQL 자체가 아니라 다음 순서다.

```text
원천 확인 -> 변환 기준 명시 -> 임시 적재 -> 키/건수/조인 검증 -> 최종 반영 -> 반복 실행 정책 확정
```

이 순서를 지키면 원천 구조가 달라도 이관 리스크를 같은 방식으로 줄일 수 있다.
