# 의료쇼핑방지망 병원환경설정 키 연계

## 배경

의료쇼핑방지망 연계에서는 환자 주민등록번호를 NIMS 공개키로 RSA 암호화한 뒤 팝업 로그인과 조회 여부 확인 흐름에 사용한다. 이때 공개키를 코드에 고정하면 병원별 가입 정보가 달라질 때 배포 없이 대응할 수 없고, 운영 환경의 설정 책임도 코드와 뒤섞인다.

## 확인된 설정 구조

- 병원환경설정은 FE의 `HisApi.getSysCfg('MS')`로 직접 조회하지 않고, 백엔드 서비스가 세션 기준으로 조회한다.
- 의쇼망 관련 값은 `MS_SYSCFG`의 `MODULE_CD='MS'`, `CTRL_CD='5'` 설정 묶음에 있다.
- 주요 키는 `AUTH_FG`, `AUTH_KEY`, `ENC_KEY`, `H`, `IP`, `PORT`, `PROD_YN`, `SWID`다.
- `ENC_KEY`는 NIMS 가입 절차에서 제공받는 암호화 공개키다.

보안상 실제 `CTD_CD` 값은 문서나 로그에 남기지 않는다.

## 적용 패턴

프론트는 환자번호처럼 화면 흐름에 필요한 최소 값만 암호화 API 요청에 전달한다. 공개키 같은 병원별 민감 설정값은 백엔드가 세션의 `CO_CD`, `loginDivCd` 기준으로 `MS_SYSCFG`에서 직접 조회한다. FE가 `HisApi.getSysCfg('MS')`로 범용 설정을 직접 조회하면 `ENC_KEY` 같은 BE 전용 값까지 같이 노출되므로, 의료쇼핑방지망 전용 옵션 API를 두고 FE-safe 응답만 내려주는 구조가 더 맞다.

백엔드는 `MedicalShoppingPreventionQuery.xml`에서 `CTRL_CD='5'` 의쇼망 설정 묶음을 통으로 조회하고, `ctrCd -> ctdCd` 맵으로 변환한 뒤 필요한 값을 꺼내 쓴다. 현재 적용 설정만 쓰도록 `APLY_DT`, `END_DT` 조건을 걸어야 같은 `CTR_CD`의 과거/미래 설정이 맵에서 임의로 덮이는 문제를 피할 수 있다.

암호화에는 `ENC_KEY`를 사용하지만 옵션 조회 API 응답에는 포함하지 않는다. 더 나아가 FE가 병원환경설정 형태 자체를 알 필요도 없으므로, FE에는 의료쇼핑방지망 서비스 컨텍스트만 내려준다. 이 컨텍스트에는 팝업 호출용 `popupUrl`만 포함한다.

환자 주민등록번호 조회와 암호화도 FE 별도 API로 노출하지 않는다. `getChkRpHist`, `getAllDrugList` 같은 외부 API 호출은 백엔드가 병원환경설정, 환자정보, 인증키, 암호화 주민번호, SWID, 기관번호를 조립해서 호출한다. 공개 API는 의미가 분명한 팝업 URL 생성, 저장 전 검증, 조회여부 확인으로 나누고, 범용 proxy나 `apiType` 분기는 외부 호출면에 남기지 않는다. 저장 전 검증도 FE가 현재 처방의 `bassEdiCd` 목록만 넘기고, 백엔드가 필수/권고/자가처방/조회여부를 판단한 뒤 `status`, `valid`, `popupRequired`, `waitPopup`, `message`, `ediCds`를 반환한다. 모듈형 URL은 코드 상수 대신 병원환경설정의 `IP`, `PORT`로 조립한다. 키가 없으면 공개키 스펙 생성이나 외부 API 호출 전에 명확히 실패시켜 병원환경설정 누락으로 볼 수 있게 한다.

## 재사용 기준

외부 연계 인증키, 공개키, 기관 식별자처럼 병원별로 달라지는 값은 다음 기준을 적용한다.

- 코드 상수보다 병원환경설정을 우선한다.
- 설정 키 이름은 DB나 실제 설정 조회 결과로 확인한다.
- 민감 값은 문서, 커밋 메시지, 로그에 남기지 않는다.
- FE가 범용 병원환경설정을 직접 조회하지 않게 하고, BE 전용 설정값은 BE 내부에 둔다.
- FE가 암호화 주민번호를 별도로 조회하거나 외부 API 파라미터를 조립하지 않게 한다.
- FE가 필수/권고/자가처방 분류를 직접 하지 않고, 현재 처방 EDI 코드 목록만 백엔드 검증 서비스에 전달한다.
- 설정 누락/형식 오류는 BE에서 구체적인 `resultMsg`로 던지고, FE는 이를 generic fallback 메시지로 덮어쓰지 않는다.
- 검증은 최소한 FE 파싱, BE 컴파일, 경로 한정 `git diff --check`로 확인한다.
